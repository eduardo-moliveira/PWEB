Privacidade e Segurança

Autor:
Eduardo Mariano de Oliveira
Tema:
Autenticação OAuth, JWT e segurança contra XSS/CSRF

Introdução

A expansão das aplicações web modernas trouxe desafios significativos para a proteção de dados dos usuários. A privacidade e a segurança deixaram de ser apenas diferenciais e tornaram-se requisitos fundamentais no desenvolvimento de software.

Nesse cenário, garantir que apenas usuários legítimos acessem informações sensíveis exige mecanismos de verificação e controle. O uso de padrões estabelecidos na indústria ajuda os desenvolvedores a construir sistemas mais resilientes contra ameaças constantes.

Autenticação e Autorização com OAuth

O protocolo OAuth 2.0 resolve o problema da delegação de acesso estruturando a comunicação entre diferentes sistemas. Em vez de o usuário fornecer sua senha diretamente a um aplicativo, o OAuth introduz a figura de um servidor terceirizado para gerenciar a autorização.

O fluxo básico definido na RFC 6749 envolve quatro partes principais.

  • Dono do Recurso O usuário que autoriza a manipulação de seus dados.
  • Cliente A aplicação que deseja o acesso.
  • Servidor de Autorização Recebe a solicitação e emite um token de acesso.
  • Servidor de Recursos Onde residem os dados protegidos consumidos pelo Cliente.

A separação dessas responsabilidades garante que o Cliente nunca tenha contato com a senha do usuário, operando sob o princípio do menor privilégio com acessos restritos a escopos específicos.

JSON Web Tokens

O token de acesso mencionado no fluxo do OAuth frequentemente utiliza o formato JSON Web Token. Suas principais características incluem uma estrutura autossuficiente e proteção criptográfica.

  1. Stateless: Não exige que o servidor guarde o estado da sessão em um banco de dados.
  2. Estrutura legível: Carrega as informações necessárias sobre o usuário em formato de texto.
  3. Assinatura digital: Garante que o conteúdo não foi adulterado.

Para mitigar a dificuldade de revogar acessos, a prática padrão é configurar o JWT com um tempo de vida muito curto e emitir simultaneamente um Refresh Token. Quando o JWT original expira, a aplicação utiliza o Refresh Token para solicitar um novo acesso silenciosamente.

Vulnerabilidades Comuns e Segurança

Mesmo com protocolos conceituados, as aplicações web continuam suscetíveis a ataques se o código não for cuidadosamente construído. A adoção de padrões resolve apenas uma parte da arquitetura de segurança.

O Cross-Site Scripting (XSS) ocorre quando um invasor consegue injetar código malicioso em uma página legítima. Um exemplo prático seria um fórum de discussões onde o sistema não filtra adequadamente o texto digitado. O atacante publica um comentário contendo um script oculto.

Quando vítimas leem a página, seus navegadores executam o script automaticamente, podendo capturar um JWT vulnerável e enviá-lo ao invasor.
